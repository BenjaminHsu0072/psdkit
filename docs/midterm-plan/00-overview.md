# 中期计划总览

## 目标

中期目标是把 PSDKit 从「8-bit 位图 PSD 读写库」推进到「自制画板可用的 `.psd` 持久化层」：

- `.psd` 作为画板文件保存格式。
- 全量支持当前画板特性：8-bit 带透明度位图图层、嵌套组、`normal` / `multiply` / `add` 三种混合模式。
- PSDKit 自己写出的 `.psd` 可以反复打开、编辑、写入，不丢失支持子集内的数据。
- 外部 PSD 打开时，如果存在不支持特性，提示并按规则降级或丢弃。
- 性能达到中等画布可用，有 benchmark 数据支撑。

## 非目标

- 不支持任意 Photoshop PSD 的无损编辑。
- 不引入私有 manifest 或私有元数据。
- 不做文字层、调整层、智能对象、图层样式编辑。
- 不做 16-bit / 32-bit / PSB。
- 不在中期实现蒙版；蒙版应在图层树、混合模式、持久化闭环稳定后单独立项。

## 兼容承诺

PSDKit 中期只承诺 **支持子集内无损**：

| 文件来源 | 目标行为 |
|----------|----------|
| PSDKit 自己写出的 PSD | 支持子集内反复打开、编辑、保存不丢数据 |
| Photoshop 写出的简单 PSD | 能导入支持子集；不支持特性进入兼容性报告 |
| 含复杂特性的外部 PSD | 尽量打开；文字、调整层、样式、智能对象等按规则降级或丢弃 |

## 里程碑

| 里程碑 | 内容 | 依赖 |
|--------|------|------|
| M1 | 支持子集与降级规则定稿 | 无 |
| M2 | 兼容性报告接入读取路径 | M1 |
| M3 | 嵌套组读写闭环 | M1, M2 |
| M4 | 三种混合模式读写与预览 | M1, M2 |
| M5 | `.psd` 持久化往返测试矩阵 | M3, M4 |
| M6 | 性能基准与第一轮优化 | M5 |

## 冻结 Gate

每个里程碑进入实现验收前必须满足对应 gate：

| Gate | 冻结内容 | 影响范围 |
|------|----------|----------|
| G1 | 支持 / 降级 / 拒绝的唯一行为矩阵 | M1 之后所有读取与测试 |
| G2 | `PSDCompatibilityReport` 的挂载方式、issue kind、severity | M2、M3、M4、Viewer |
| G3 | PSD layer record 顺序与内存树转换样例 | M3、M5 |
| G4 | 三种 blend mode 的 PSD key 与未知模式降级行为 | M4、M5 |
| G5 | 性能基线环境、重复次数、统计口径、阈值 | M6 |

## 建议排期

| 周期 | 主线 |
|------|------|
| 第 1 周 | 完成支持子集文档、兼容性报告 API 草案、fixture 设计 |
| 第 2-3 周 | 实现嵌套组读写与 Viewer 展示 |
| 第 3-4 周 | 实现三种混合模式读写、预览合成与 Photoshop 验证 |
| 第 5 周 | 建立反复打开/编辑/保存测试矩阵 |
| 第 6 周 | 建立 benchmark，优化合成、平面转换、RLE 热路径 |

实际排期应以 `swift test`、Photoshop 手工验证和性能数据为准，而不是以功能写完为准。

## 验收总标准

中期计划完成时，必须满足：

1. PSDKit 可新建一个含嵌套组、透明图层、三种混合模式的 PSD。
2. 该 PSD 可被 Photoshop 打开，图层结构和混合模式显示正确。
3. 该 PSD 可被 PSDKit 打开、编辑、保存，再打开后树结构、图层像素、可见性、不透明度、混合模式一致。
4. 外部 PSD 中的不支持特性不会静默误导用户，必须进入兼容性报告。
5. benchmark 覆盖小/中/压力三档文档，并记录打开、保存、预览合成耗时和峰值内存。
